Project Overview

This project focuses on analyzing transactional data to identify patterns and anomalies associated with fraudulent activity. 
The goal is to uncover high-risk behaviors and provide actionable insights to improve fraud detection and risk mitigation strategies.

Dataset

~300,000 transaction records
Includes features such as transaction amount, time of day, and customer behavior
Optimized dataset used after removing unnecessary columns for efficient analysis

Tools & Technologies

Excel – Data cleaning and preprocessing
Power BI – Data visualization and dashboard creation
DAX – Calculated columns and measures for KPI tracking

Key Analysis Performed

Data cleaning and transformation of large transactional dataset
Exploratory Data Analysis (EDA) to identify fraud patterns
Segmentation of transactions based on amount and time of day
Analysis of fraud rate across different spend ranges
Time-based analysis to detect high-risk periods

Key Insights

Fraud rate is significantly higher for high-value transactions during nighttime (~10x higher than average)
Higher transaction amounts show increased fraud probability
Certain time windows exhibit elevated fraud activity, indicating potential behavioral patterns

Business Impact

Enables targeted fraud monitoring during high-risk time periods
Identify high-value transactions requiring additional verification
